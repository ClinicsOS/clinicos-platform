# ClinicOS — الواجهة الأمامية (Frontend)

واجهة نظام إدارة العيادات ClinicOS، مبنية بـ **Next.js 14 + TypeScript + Tailwind**، وموصولة بالباك اند الذي بنيناه (Express + MongoDB).

---

## ✅ المتطلبات قبل التشغيل

1. **Node.js 18 أو أحدث** (عندك جاهز).
2. **الباك اند شغّال** على البورت 5000:
   - افتح ترمينال داخل مجلد `server`
   - نفّذ: `npm run dev`
   - تأكد من ظهور: `🚀 ClinicOS API running` و `✅ MongoDB connected`

---

## 🚀 خطوات التشغيل (مرة واحدة فقط)

1. فك ضغط المجلد وضع `client` بجانب مجلد `server` (داخل مجلد `clinicos`).
2. افتح ترمينال **داخل مجلد `client`**:
   ```
   cd client
   ```
3. نزّل المكتبات (يأخذ دقيقة إلى دقيقتين):
   ```
   npm install
   ```
4. شغّل الواجهة:
   ```
   npm run dev
   ```
5. افتح المتصفح على:
   ```
   http://localhost:3000
   ```

> ملف `.env.local` جاهز مسبقاً ويشير إلى `http://localhost:5000/api`. لو غيّرت بورت الباك اند، عدّل هذا الملف.

## 🗺️ خريطة الصفحات

| الرابط | الصفحة |
|---|---|
| `/` | Splash السينمائي (المكعب يدور ← يصغر ← يطير للزاوية) ثم صفحة الهبوط |
| `/signup` | تسجيل عيادة جديدة (مع معاينة رابط الحجز الحيّة) |
| `/signin` | تسجيل الدخول |
| `/dashboard` | لوحة التحكم — أرقام اليوم الحيّة + رسم الأسبوع + جدول اليوم |
| `/appointments` | تقويم أسبوعي (سبت–خميس) بألوان الأطباء + إنشاء موعد |
| `/patients` | قائمة المرضى مع بحث حيّ + إضافة مريض |
| `/patients/profile` | ملف المريض (أعلى جرعة 3D: مدارات، بطاقات عمق، رصيد عائم) |
| `/invoices` | الفواتير + إنشاء فاتورة + تسجيل دفعات |
| `/settings` | بيانات العيادة وساعات الدوام + رابط الحجز + إدارة الموظفين |
| `/book/{slug}` | **صفحة الحجز العامة للمريض** — بدون حساب |

> رابط الحجز الخاص بعيادتك موجود دائماً أسفل القائمة الجانبية وفي الإعدادات (زر نسخ).

## 🌗 الوضع الليلي واللغة

- **الثيم**: زر الشمس/القمر أعلى يمين لوحة التحكم — الاختيار يُحفظ تلقائياً. الافتراضي هو الوضع الغامق.
- **اللغة**: مبدّل EN/AR بجانبه — الواجهة تنقلب RTL بالكامل مع العربية.

## 🧪 مسار فحص سريع (5 دقائق)

1. افتح `/` وشاهد حركة المكعب ← تنكشف صفحة الهبوط.
2. سجّل الدخول بحساب المالك (ahmad@alshifa.jo).
3. الداشبورد: يجب أن تظهر أرقام اليوم الحقيقية من الباك اند.
4. المرضى: اكتب في البحث وراقب الفلترة الحيّة.
5. الإعدادات ← انسخ رابط الحجز ← افتحه في تبويب جديد.
6. احجز موعداً كمريض ← يظهر الرقم المرجعي BK-XXXXX.
7. ارجع للمواعيد في اللوحة ← الحجز الجديد ظاهر في التقويم.

## 🧰 ملاحظات تقنية

- **التوكن**: يُخزَّن عبر Zustand persist ويُرفق تلقائياً بكل طلب (axios interceptor).
- **بيانات السيرفر**: React Query مع تحديث تلقائي كل دقيقة للداشبورد.
- **التواريخ**: تُعرض بتوقيت UTC المطابق لطريقة تخزين الباك اند (توقيت العيادة).
- **ملف المريض**: يُفتح من قائمة المرضى (يُمرَّر عبر session store). إضافة endpoint
  `GET /api/patients/:id` لاحقاً في الباك اند ستجعل فتح الملف برابط مباشر ممكناً.
- لوحة **Super Admin** مؤجلة باتفاقنا — ستُبنى كتطبيق منفصل.
